# AI Automation Agency — Claude Code Skill

> A Claude Code skill that positions Claude as a senior automation consultant — structured thinking, real deliverables, and a methodology that mirrors how professional agencies actually operate.

## What This Does

Most people use Claude for ad-hoc automation advice. This skill changes that. It loads a complete consulting framework into Claude Code — a six-phase workflow from problem diagnosis through client delivery — so you get structured, professional output instead of generic suggestions.

Instead of "here's how you could automate that," you get diagnosis, discovery, stack selection, architecture design, and client-ready documentation. The same process a professional automation agency follows on a paid engagement.

## Core Workflow

| Phase | What Happens |
|---|---|
| **Diagnosis** | Find the real problem, not the stated one. Quantify the cost before touching any tool. |
| **Discovery** | Map triggers, inputs, outputs, technical stacks, volume, and stakeholders. |
| **Stack Selection** | Evaluate n8n, Make.com, Zapier, and custom builds based on actual requirements — not tool preference. |
| **Architecture** | Design complete flows with decision branches, error handling, and failure scenarios. |
| **Building** | Implementation with retries, logging, and test cases. |
| **Delivery** | Client-ready documentation, handover materials, and recommendations for future phases. |

## What's Included

- **Skill definition** — the full consulting framework loaded into Claude Code
- **Pattern library** — 8 documented automation patterns (webhook routing, lead qualification, CRM sync, and more)
- **Stack selection guide** — structured comparison of major platforms with decision criteria
- **Sample deliverables** — complete output from a hotel booking system engagement (architecture, docs, handover)

## Setup

1. Copy `skill/SKILL.md` into `.claude/skills/ai-automation-agency/` in your Claude Code project
2. Claude will apply the consulting framework automatically when you discuss automation problems

## Use Cases

- **Automation consultants** — run client engagements with a structured methodology instead of reinventing the approach each time
- **Agency owners** — give junior staff a framework that produces professional, consistent output from day one
- **In-house teams** — use the diagnosis and discovery phases before building anything; they surface requirements that would otherwise be missed mid-project
- **Freelancers** — deliver client-ready documentation and architecture diagrams, not just working code
